# 스토리 관리

[English](managing-stories.md) | **한국어**

> 영문이 기준입니다: [managing-stories.md](managing-stories.md).

혼란을 없애는 규칙 하나:

> **관리는 하네스 루트에서. 플레이는 스토리 폴더에서.**
> 한 스토리 = 한 폴더 = 한 Claude Code 세션.

## 어디에 무엇이

| 위치 | 무엇 |
|---|---|
| 클론한 레포(하네스 루트) | 엔진: 스킬, 템플릿, `scripts/`. 관리 명령은 여기서. |
| `examples/imperial-ball/` | 번들 **데모** — 샘플이지 네 세이브 아님. |
| `stories/<이름>/` | **네** 스토리들 (git 무시). 각각이 플레이 가능한 한 폴더. |

(파워유저: `STORY_HOME`을 설정해 레포 밖에 보관 가능.)

## 라이프사이클

```bash
# 1. 생성 — 하네스 루트에서
/new-story 우주오페라             # → stories/우주오페라/ 생성

# 2. 플레이 — 스토리 폴더에서
cd stories/우주오페라 && claude
# GM 보이스 + 자동저장 자동 활성 (그 폴더의 .claude/settings.json)

# 3. 목록 — 하네스 루트에서
/stories

# 4. 재개 — 그 폴더 다시 열면 됨; 상태는 파일에 있음
cd stories/우주오페라 && claude
```

## 스토리 전환

다른 폴더를 연다. 스토리는 디렉터리에 묶여 있어서 "전환" = 다른 스토리 폴더에서 세션 시작. 앱 내
picker/launcher는 [로드맵](../../ROADMAP.md) P3(로컬 웹)에서 옴.

## 왜 이렇게

스토리는 평범한 파일이라 **폴더가 곧 세이브** — 휴대 가능, git 버전관리, 들여다보기 가능.
스토리를 `examples/`가 아니라 `stories/`에 두면 네 콘텐츠가 배포물과 분리돼 각 스토리가 깔끔한
독립 단위가 됨.
